

## שיפור מסך ה-Empty State בדשבורד למשתמשים חדשים

### מצב נוכחי
כשמשתמש חדש נרשם ועדיין לא הוסיף ילדים, הוא רואה כרטיס מינימלי עם אייקון אפור, הטקסט "אין ילדים רשומים" וכפתור "הוסף ילד". זה לא מזמין ולא מסביר מה עליו לעשות.

### עיצוב מוצע
מסך קבלת פנים חם ומזמין שמנחה את המשתמש בשלושה שלבים ברורים:

```text
┌─────────────────────────────────────────────┐
│                                             │
│              🛡️ (אייקון מגן גדול)           │
│                                             │
│         ברוכים הבאים ל-Kippy! 👋            │
│                                             │
│    כדי להתחיל להגן על הילדים שלכם,          │
│    יש להוסיף ילד ולחבר את המכשיר שלו        │
│                                             │
│  ┌───────┐  ┌───────┐  ┌───────┐            │
│  │  ①    │  │  ②    │  │  ③    │            │
│  │הוסיפו │  │חברו   │  │קבלו  │            │
│  │ ילד   │  │מכשיר  │  │התראות │            │
│  └───────┘  └───────┘  └───────┘            │
│                                             │
│        [ + הוסיפו את הילד הראשון ]          │
│                                             │
│     זה לוקח פחות מדקה ⏱️                    │
│                                             │
└─────────────────────────────────────────────┘
```

### שינויים בקוד

**קובץ: `src/pages/Dashboard.tsx`** — שורות 774-781

החלפת הבלוק הנוכחי של ה-empty state בעיצוב חדש שכולל:

1. **כותרת מזמינה** — "ברוכים הבאים ל-Kippy! 👋"
2. **משפט הסבר קצר** — "כדי להתחיל להגן על הילדים שלכם ברשת, הוסיפו ילד וחברו את המכשיר שלו"
3. **3 שלבים ויזואליים** (אייקונים + טקסט קצר):
   - שלב 1: "הוסיפו ילד" (אייקון UserPlus)
   - שלב 2: "חברו מכשיר" (אייקון Smartphone)
   - שלב 3: "קבלו התראות" (אייקון Shield)
4. **כפתור CTA בולט** — "הוסיפו את הילד הראשון"
5. **טקסט מעודד** — "זה לוקח פחות מדקה ⏱️"

כל זה בתוך כרטיס אחד עם רקע gradient עדין של primary, אנימציית fade-in עם framer-motion, ושמירה על השפה העיצובית הקיימת (dark theme, RTL).
