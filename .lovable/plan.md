

## תוכנית: הצגת שם האדמין בלוג פעילות

### מצב נוכחי
לוג הפעילות (`admin_activity_log`) שומר `admin_user_id` אבל ב-UI מוצג רק סוג הפעולה + זמן — **לא מוצג מי ביצע את הפעולה**.

### שינוי מתוכנן

**קובץ: `src/pages/admin/AdminCustomerProfile.tsx`**

1. **Fetch שמות אדמינים** — אחרי שליפת הלוג, לאסוף את כל ה-`admin_user_id` הייחודיים ולשלוף את השמות שלהם מטבלת `parents`
2. **הצגה ב-UI** — הוספת שם האדמין בכל שורת לוג, לפני הזמן

```text
שורת לוג נוכחית:
┌──────────────────────────────────────────┐
│ [התחזות]  חודש Premium ל-2 ילדים  לפני 3 שעות │
└──────────────────────────────────────────┘

שורת לוג חדשה:
┌──────────────────────────────────────────────────────┐
│ [התחזות]  חודש Premium ל-2 ילדים  yariv · לפני 3 שעות │
└──────────────────────────────────────────────────────┘
```

### פרטים טכניים

- שליפת שמות: query ל-`parents` עם `id.in(uniqueAdminIds)` — לא צריך migration
- מיפוי `Map<string, string>` של `admin_user_id → full_name`
- fallback: אם אין שם → הצגת "אדמין" גנרי
- אין צורך בשינוי DB — השדה `admin_user_id` כבר קיים
