

# הוספת חיווי אדום בולט לכרטיס ילד עם מכשיר מנותק

## המטרה
כרטיס של ילד עם מכשיר מנותק יקבל **מסגרת אדומה בהירה + באנר עליון אדום** שלא ניתן לפספס — באותו דפוס בדיוק של חיווי "הגבלה פעילה" (כתום) שכבר קיים.

## מה משתנה — `src/components/home-v2/ChildCardV2.tsx`

### 1) מסגרת הכרטיס (שורה 112)
לוגיקה דו-מצבית עם **עדיפות לניתוק** (החמור יותר):
- מכשיר מנותק → `border-red-300` + `ring-1 ring-red-200` (לעדינות נוספת)
- הגבלה פעילה → `border-amber-300` (כמו היום)
- ברירת מחדל → `border-gray-200`

הניתוק מקבל עדיפות חזותית גם אם יש במקביל הגבלה — כי "מכשיר מנותק" מבטל למעשה את כל שאר הסטטוסים (ההגבלה לא נאכפת אם המכשיר לא מדווח).

### 2) באנר עליון אדום (חדש, לפני באנר ההגבלה הקיים)
```
⚠ המכשיר לא מחובר — ייתכן שהשליטה אינה פעילה
```
- רקע: `bg-red-50`
- גבול תחתון: `border-red-200`
- אייקון: `WifiOff` (lucide) בצבע `text-red-600`
- טקסט: `text-red-700` עם `font-semibold`
- אם יש גם הגבלה פעילה — שני הבאנרים יוצגו זה מתחת לזה (אדום למעלה, כתום מתחתיו).

### 3) חיזוק הנקודה הצבעונית הקיימת בכותרת (שורה 134)
היום הנקודה האפורה (`bg-gray-300`) מציינת "לא מחובר" — שינוי ל-`bg-red-500` כדי להיות עקבי עם הצבע החדש. הטקסט "לפני 2 ימים" יישאר אפור (לא צריך להעמיס).

### 4) שימוש בסף ניתוק קיים
ממשיך להשתמש ב-`isConnected()` הקיים (סף 24 שעות) — שמירה על עקביות עם הספירה "1 מנותק" שכבר מופיעה ב-`FamilyStatusHero`. אם בעתיד נרצה להחמיר ל-1 או 3 שעות — זה שינוי גלובלי נפרד.

## יבוא נוסף נדרש
```ts
import { ..., WifiOff } from "lucide-react";
```

## מה לא נוגעים
- אין שינוי ב-`FamilyStatusHero` — הספירה כבר תקינה.
- אין שינוי ב-`HomeV2.tsx` — הלוגיקה כבר מועברת בתוך `child.device.last_seen`.
- אין שינוי ב-V1 / DB / הרשאות / Android.
- אין שינוי בלוגיקת ה-isConnected עצמה.

## תוצאה צפויה
- כרטיס רואי בולט מיד עם מסגרת אדומה + באנר אזהרה ברור.
- כרטיס יריב (מחובר) נשאר נקי כמו היום.
- אם בעתיד יהיה לרואי גם הגבלה פעילה — שני באנרים יוצגו, הניתוק קודם.
- ההורה לא יכול לפספס שהמכשיר לא מחובר.
