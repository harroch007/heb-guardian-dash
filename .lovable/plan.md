

# הצגת חיווי "חסום — חרג ממגבלת זמן מסך" בכרטיס הילד

## הבעיה
כרגע ב-`ChildCardV2` יש חיווי כתום ("הגבלה פעילה") רק עבור הגבלות מתוזמנות — שינה / לימודים / שבת (מבוסס על `activeRestriction` שמגיע מ-`schedule_windows`). כשהילד **חורג ממגבלת זמן המסך היומית** (`daily_screen_time_limit_minutes`) המכשיר ננעל בפועל באנדרואיד, אבל בדשבורד אין שום אינדיקציה — ההורה רואה כרטיס "רגיל".

בצילום שהעלית רואים את יריב: 2:33 שעות שימוש, 0 דק׳ נותרו — כלומר חרג. אין שום באנר/מסגרת שמראה שהוא חסום.

## מה צריך להוסיף

### 1) זיהוי מצב "חסום עקב חריגה" ב-`HomeV2.tsx`
לכל ילד לחשב flag חדש `screenTimeExceeded`:
```
screenTimeExceeded = 
  daily_limit > 0 
  AND (totalUsageToday + bonusToday) >= (daily_limit + bonusToday)
  AND !activeRestriction  // אם יש שינה/שבת זה גובר ויראה כתום
```
או פשוט יותר:
```
screenTimeExceeded = daily_limit > 0 AND remainingMinutes <= 0
```
(הערך `remainingMinutes` כבר מחושב — זה ה-"0 דק׳ נותר" שמופיע בכרטיס.)

הערך יועבר כ-prop חדש ל-`ChildCardV2`.

### 2) חיווי ויזואלי ב-`ChildCardV2.tsx` — עקבי עם הקיים
היררכיית עדיפות מסגרת (החמור גובר):
1. **מנותק** → אדום (`border-red-300 ring-red-200`) — קיים
2. **חסום עקב חריגת זמן מסך** → אדום בהיר (`border-red-300`) — **חדש**
3. **הגבלה מתוזמנת פעילה** (שינה/לימודים/שבת) → כתום — קיים
4. ברירת מחדל → אפור

### 3) באנר עליון חדש — אדום
בדומה לבאנר "המכשיר לא מחובר" שיצרנו, אבל עם תוכן וצבע מותאמים:
- אייקון: `Lock` או `ShieldAlert` (lucide)
- רקע: `bg-red-50`, גבול: `border-red-200`
- טקסט: **"המכשיר נעול — הילד חרג ממגבלת זמן המסך היומית"**
- צבע טקסט: `text-red-700 font-semibold`

אם הילד גם **מנותק** וגם חרג — באנר הניתוק גובר (כי אם מנותק, אין באמת אכיפה).

### 4) שינוי הצבע של הצ׳יפ "0 דק׳ נותר"
כיום כתוב "0 דק׳" בכתום. במצב חריגה הצ׳יפ יהפוך לאדום (`text-red-600`) כדי שיהיה אחיד עם הבאנר והמסגרת.

## דוגמה ויזואלית (יריב אחרי השינוי)
```
┌─────────────────────────────────────────┐  ← border-red-300
│ 🔒 המכשיר נעול — חרג ממגבלת זמן המסך היומית│  ← bg-red-50
├─────────────────────────────────────────┤
│  י  יריב                          100% 🔋│
│     ● מחובר עכשיו                       │
│  📱 30 דק׳   ⏱ 0 דק׳ (אדום)  ⏰ 2:33 שעות │
│      בנק בונוס   נותר          זמן מסך   │
│  📍 גדעון האוזנר, הרצליה                 │
└─────────────────────────────────────────┘
```

## מה לא נוגעים
- אין שינוי באכיפה עצמה (אנדרואיד) — היא כבר עובדת.
- אין שינוי ב-DB / RPC.
- אין שינוי ב-`FamilyStatusHero` — אם בעתיד נרצה למנות "ילדים חסומים" זה שינוי נפרד.
- אין שינוי ב-V1.
- לוגיקת `activeRestriction` הקיימת (שינה/שבת) נשארת בדיוק כמו שהיא — היא עדיין מציגה כתום.

## קבצים שיעודכנו
- `src/pages/HomeV2.tsx` — חישוב `screenTimeExceeded` לכל ילד והעברתו כ-prop.
- `src/components/home-v2/ChildCardV2.tsx` — באנר אדום חדש, עדכון `borderClass`, צבע אדום לצ׳יפ "0 נותר".

## תוצאה צפויה
- ההורה רואה מיד שיריב חסום עקב חריגה — באנר אדום בולט + מסגרת אדומה.
- אם הילד גם מנותק → באנר ניתוק (ניתוק חמור יותר).
- אם הילד גם בשעת שינה → גם הכתום וגם האדום עלולים להיות רלוונטיים, אך החריגה גוברת חזותית כי היא מצב פעיל של "ננעל" (אם בשינה ממילא חסום ואין משמעות לחריגה — נציג רק את הכתום של השינה).
