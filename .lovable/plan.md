

# ניקוי DeviceHealthBanner ממרכיבי ניטור וואטסאפ

## מה רלוונטי ומה לא — כשניטור הוואטסאפ כבוי

הבאנר מציג כיום 7 הרשאות + תג "ניטור פעיל/לקוי". מתוכם, חלק נוגעים בלעדית לוואטסאפ והשאר חיוניים לאכיפה הבסיסית של Kippy.

| רכיב | תלוי בוואטסאפ? | החלטה |
|---|---|---|
| תג "ניטור פעיל / ניטור לקוי" (MessageCircle) | כן — מחושב מ-accessibility + notificationListener | **להסתיר** |
| הרשאת "שירות נגישות" | בעיקר לוואטסאפ | **להסתיר** |
| הרשאת "האזנה להתראות" | בעיקר לוואטסאפ | **להסתיר** |
| סטטיסטיקת שימוש | זמן מסך, אכיפת מגבלות | **להשאיר** |
| מיקום | איתור הילד | **להשאיר** |
| שירותי מיקום (GPS) | איתור הילד | **להשאיר** |
| אופטימיזציית סוללה | שמירה שהאפליקציה רצה | **להשאיר** |
| הצגה מעל אפליקציות | מסך חסימה | **להשאיר** |
| דגם מכשיר + גרסה + זמן דיווח | אבחון כללי | **להשאיר** |

## הפתרון
כש-`WHATSAPP_MONITORING_ENABLED === false`:
1. להסתיר את תג "ניטור פעיל/לקוי" בכותרת.
2. לסנן את `accessibilityEnabled` ו-`notificationListenerEnabled` מרשימת ההרשאות (גם בספירת "X הרשאות חסרות").

## שינוי טכני
קובץ יחיד: `src/components/controls/DeviceHealthBanner.tsx`
- ייבוא `WHATSAPP_MONITORING_ENABLED` מ-`@/config/featureFlags`.
- סינון של מפתחות הוואטסאפ מ-`allPermissions` ומחישוב `missingPermissions` כאשר ה-flag כבוי.
- הסתרה תנאית של ה-`Badge` של "ניטור פעיל/לקוי".

## מה לא נוגעים
- אין שינוי ב-`ProblemBanner.tsx` (הוא יטופל בנפרד אם תרצה).
- אין שינוי ב-DB / Android / hook `useChildControls`.
- כשנפעיל את `WHATSAPP_MONITORING_ENABLED=true` הכל יחזור אוטומטית.

## אימות
- מסך `/child-v2/:id` → רואים רק 5 הרשאות (במקום 7), ללא תג "ניטור פעיל".
- אם כל 5 ההרשאות הרלוונטיות מאושרות → הכותרת "כל ההרשאות פעילות" בירוק.
- אם חסרה אחת מהן → "1 הרשאות חסרות" + רק היא ברשימה.
