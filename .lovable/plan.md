

## הוספת הסברים והנחיות תיקון להרשאות במסך בריאות המכשיר

### מה ישתנה

**קובץ: `src/components/controls/DeviceHealthBanner.tsx`**

1. **מילון הסברים** — לכל הרשאה נוסיף שדה `description` (הסבר קצר) ו-`fix` (הנחיות תיקון):
   - שירות נגישות → "מאפשר לזהות אפליקציות פתוחות ולחסום תוכן" / "הגדרות → נגישות → Kippy → הפעל"
   - האזנה להתראות → "מאפשר לנטר הודעות נכנסות לצורך ניתוח בטיחות" / "הגדרות → התראות → גישה להתראות → Kippy"
   - וכו' לכל 7 ההרשאות

2. **אייקון סימן שאלה (?)** — ליד כל הרשאה (פעילה או חסרה) יופיע אייקון `HelpCircle` קטן. לחיצה עליו תפתח/תסגור את ההסבר מתחת לשורה (toggle).

3. **כפתור "מה לעשות?"** — יופיע **רק** ליד הרשאות חסרות. לחיצה תציג את הנחיות התיקון הספציפיות.

4. **כשהכל תקין** — המסך נשאר נקי כמו היום (רק "כל ההרשאות פעילות"). סימני שאלה יופיעו רק אם ההורה ירצה להבין מה כל הרשאה עושה.

### מבנה UI

```text
┌─ הרשאה חסרה ─────────────────────────┐
│ ⚠ שירות נגישות  (?) [מה לעשות?]      │
│   ↳ מאפשר לזהות אפליקציות פתוחות...  │  ← נפתח בלחיצה על ?
│   ↳ הגדרות → נגישות → Kippy → הפעל   │  ← נפתח בלחיצה על "מה לעשות?"
└───────────────────────────────────────┘

┌─ הרשאה פעילה ─────────────────────────┐
│ ✓ שירות נגישות  (?)                    │
│   ↳ מאפשר לזהות אפליקציות פתוחות...  │  ← רק בלחיצה על ?
└───────────────────────────────────────┘
```

### שינוי עיצובי
- הרשימה תעבור מ-badges בשורה אחת → לרשימה אנכית עם שורה לכל הרשאה
- הרשאות חסרות: רקע `warning/10`, טקסט `warning`
- הרשאות פעילות: רקע `success/10`, טקסט `success`
- כפתור "מה לעשות?" קטן בסגנון `ghost` עם אייקון `Wrench`

### State
- `expandedInfo: Set<string>` — אילו הרשאות ההסבר שלהן פתוח
- `expandedFix: Set<string>` — אילו הרשאות ההנחיות שלהן פתוחות
